## Definition of Done

### Задание 1. Разработка функционала с применением библиотеки потоковой обработки данных.
- Система обработки потоков сообщений с функциями блокировки пользователей и цензуры сообщений
    - Поток обработки сообщений в котором пользователи смогут блокировать других пользователей
    - Цензурирование забаненных слов в сообщениях

### Задание 2. Анализ и агрегирование сообщений с помощью ksqlDB
- Агрегация данных:
    - `messages_stream` - стрим данных из топика `messages`
    - `total_messages` - таблица, показывающая общее количество отправленных сообщений всеми пользователями
    - `unique_recipients` - таблица, показывающая количество уникальных получателей сообщений
    - `user_statistics` - таблица агрегирования данных по каждому пользователю:
        - `qty_messages` - кол-во сообщений, отправленных каждым пользователем
        - `unique_recipients` - кол-во уникальных получателей каждого сообщения

## Application Launch

1. Скачать репозиторий, установить requirements из `requirements.txt`
2. Запустить Kafka-cluster через terminal:
```bash
docker compose up -d
```
3. Создать топики запустив `run_admin.py`
4. Запустить Faust приложение для задания 1 `message_filter_app.py` :
    - фильтрует сообщения пользователей:
        - если отправитель заблокирован получателем, то получатель не получит это сообщение -> сообщение отправится в топик `blocked_messages`
        - если отправитель НЕ заблокирован получателем, то сообщение дойдет до адресата -> сообщение отправится в топик `filtered_messages`
5. Запустить Producer из: `run_producer_task_1.py` чтобы проверить логику работы приложения для задания 1.

Все сообщения поступают в топик `user_events`, дальше в зависимости от `event_type`:
- `event_type == message` - сообщение либо доходит до адресата (отправляется в топик `filtered_messages`), либо не доходит до адресата (отправляется в топик `blocked_messages`)

- `event_type == block` - блокировка отправителя получателем
- `event_type == unblock` - разблокировка отправителя получателем
- Шаги проверки маршрутизации сообщений:
    - `00` - Тестовое сообщение -> видим в user_events
    - `01` - Сообщение от NOT blocked user -> filtered_messages
    - `02` - блокируем пользователя
    - `03` - Сообщение от blocked user -> blocked messages
    - `04` - разблокируем пользователя
    - `05` - Сообщение от unblocked user -> filtered messages

- Шаги проверки цензуры сообщений:
    - `06` - добавляем слово в список запрещенных
    - `07` - отправляем сообщение с запрещенным словом -> будет заменено на `*censored*` у получателя
    - `08` - убираем слово из списка запрещенных
    - `09` - отправляем сообщение -> нет цензуры

6. Запустить Producer из: `run_producer_task_2.py` для отправки данных в топик `messages` - Задание 2
